命令

init

  • 描述：创建一个 commit ，commit message 默认为 initial commit ，branch 默认为 master ，时间戳默认为 00:00:00 UTC, Thursday, 1 January 1970 。
  • 运行时间：O(1)
  • 错误
    • 如果当前文件夹已存在 Gitlet 控制，输出错误信息 A Gitlet version-control system already exists in the current directory.
  • 代码行数：~15

add [file name]

  • 描述：将一个当前文件的复制加入 staging area。如果该文件已经存在于 staging area，则直接覆盖旧版本。staging area 存在于 .gitlet 中。如果该文件与前一个 commit 的版本完全一致，则不将其加入 staging area 并将该文件从 staging area 中去除。
  • 运行时间：最坏情况与被加入文件大小成线性，O(lg N)其中 N 为 commit 的文件数
  • 错误
    • 如果文件不存在，输出错误信息 File does not exist.
  • 代码行数：~20

commit [message]

  • 描述：
    • 创建一个新的 commit，其储存的文件初始与上一个 commit 一致，在此基础上增加更新内容，时间戳和信息也有所不同；
    • 在 commit 完成之后，清空 staging area；
    • commit 不对工作仓库中的内容进行任何更改，只更改 .gitlet ，因此在 add 或 rm 之后，对文件本身的更改不影响下一次 commit ；
    • 新的 commit 应该加在 commit树 上；
    • 新的 commit 变为最新的 commit，并且 head 指针指向它；
    • commit 必须包含日期和时间元信息；
    • commit 包含被用户指定的日志消息，在命令中只占一个 args参数;
    • commit 通过 SHA-1 id辨别。
  • 运行时间：不劣于与跟踪文件总大小的线性关系
  • 运行空间：不应增长多于跟踪文件的总大小（使用 Blob 和 SHA-1 跟踪文件，而非再复制一份）
  • 错误
    • 如果没有文件在 staging area，输出错误信息 No changes added to the commit.
    • 如果输入的日志消息不是非空字符串，输出错误信息 Please enter a commit message.
  • 代码行数：~35

rm [file name]

  • 描述：
    • 如果该文件处于 staging area，将该文件从 staging area 中移除；
    • 如果该文件在之前的 commit 中已被跟踪，则标记为删除，并在用户尚未删除它的情况下将该文件从工作仓库中移除。
  • 运行时间：O(1)
  • 错误
    • 如果文件既没有被暂存也没有被 head 指向的 commit 跟踪，输出错误信息 No reason to remove the file.
  • 代码行数：~20

log

  • 描述：
    • 从当前 head 指向的 commit 开始，显示每一个 commit 的信息，直到 initial commit；
    • 格式

      ===
commit a0da1ea5a15ab613bf9961fd86f010cf74c7ee48
Date: Thu Nov 9 20:00:05 2017 -0800
A commit message.

===
commit 3e8bf1d794ca2e9ef8a4007275acf3751c7170ff
Date: Thu Nov 9 17:01:33 2017 -0800
Another commit message.

===
commit e881c9575d180a215d1a636545b8fd9abfb1d2bb
Date: Wed Dec 31 16:00:00 1969 -0800
initial commit
      ===
commit 3e8bf1d794ca2e9ef8a4007275acf3751c7170ff
Merge: 4975af1 2c1ead1
Date: Sat Nov 11 12:30:00 2017 -0800
Merged development into master.
  • 运行时间：与 head 的历史 commit 数量成线性
  • 错误：无
  • 代码行数：~20

global-log

  • 描述：以类似于 log 的格式显示所有的历史 commit ，无需顺序。（使用 gitlet.Utils 中的方法）
  • 运行时间：与历史 commit 数量成线性
  • 错误：无
  • 代码行数：~10

find [commit message]

  • 描述：显示所有 commit message 与指定消息相同的 commit 的 id，每个一行。（使用 gitlet.Utils 中的方法）
  • 运行时间：与历史 commit 数量成线性
  • 错误
    • 如果没有这样的 commit 存在，输出错误信息 Found no commit with that message.
  • 代码行数：~15

status

  • 描述：
    • 显示当前存在的所有 branch，使用 * 标记当前 branch；
    • 显示暂存的文件，每个文件名一行；
    • 显示移除的文件，每个文件名一行；
    • 显示所有的未暂存更改文件，括号内显示被删除或被修改，每个文件名一行。未暂存更改指的是：
      • 在 current commit 中被跟踪，之后被更改，但是没有被暂存；
      • 已经被暂存，但是暂存内容与工作仓库内容不同；
      • 已经被暂存，但是已在工作仓库中被删除；
      • 在 current commit 中被跟踪，没有被标记为删除，但是已在工作仓库中被删除。
    • 显示所有的未跟踪文件，每个文件名一行。未跟踪指的是：
      • 之前未跟踪的文件没有被加入 staging area；
      • 被标记为删除的文件在 Gitlet 不知情的情况下被重新创建。
    === Branches ===
*master
other-branch
  
=== Staged Files ===
wug.txt
wug2.txt
  
=== Removed Files ===
goodbye.txt
  
=== Modifications Not Staged For Commit ===
junk.txt (deleted)
wug3.txt (modified)
  
=== Untracked Files ===
random.stuff
  • 运行时间：仅依赖于工作仓库中的数据、staging area 中文件数量以及 branch 总数
  • 错误：无
  • 代码行数：~45

checkout

  • 使用：
    1. checkout -- [file name]
    1. checkout [commit id] -- [file name]
    1. checkout [branch name]
  • 描述：
    1. 将 head commit 中的对应文件移动到工作仓库中，若已存在同名文件则覆写，不暂存新版本的文件。
    1. 将对应 id 的 commit 中的对应文件移动到工作仓库中，若已存在同名文件则覆写，不暂存新版本的文件； [commit id] 需要支持40个字符以下的缩写。
    1. 将给出 branch 的 commit 中的所有文件移动到工作仓库中，覆写已存在的同名文件；将给出的 branch 作为当前分支（HEAD）；所有在当前分支跟踪的、但在指定 branch 没有被跟踪的文件都被删除；除非指定 branch 是当前 branch，否则清空 staging area。
  • 运行时间：
    1. 与指定文件大小成线性
    1. 与指定 commit 文件总大小成线性；
      与
      commit 总数和 branch 总数成常数关系（使用 HashMap）；
  • 错误
    1. 如果指定 id 的 commit 不存在，输出错误信息 No commit with that id exists.
    1. 如果该文件在指定 commit 中不存在，输出错误信息 File does not exist in that commit.
    1. 如果指定 id 的 branch 不存在，输出错误信息 No such branch exists. ；如果指定的 branch就是当前 branch，输出 No need to checkout the current branch.；如果一个工作中文件还未被跟踪、会被 checkout 操作覆写，输出 There is an untracked file in the way; delete it, or add and commit it first.
  • 代码行数：
    1. ~15
    1. ~5
    1. ~15

branch [branch name]

  • 描述：新建一个以给定名称命名的 branch，将其指向 head commit，不立即切换到新建分支。
  • 运行时间：O(1)
  • 错误
    • 如果已存在同名分支，输出错误信息 A branch with that name already exists.
  • 代码行数：~10

rm-branch [branch name]

  • 描述：删除指定名称的 branch。
  • 运行时间：O(1)
  • 错误
    • 如果该名称分支不存在，输出错误信息 A branch with that name does not exist.
    • 如果要删除的分支是当前分支，输出错误信息 Cannot remove the current branch.
  • 代码行数：~15

reset [commit id]

  • 描述：
    • 将所有给定 commit 跟踪的文件版本取出；
    • 移除所有未在指定 commit 中出现的已跟踪文件；
    • 将当前分支的 head 移动到指定 commit 上；
    • 与 checkout 同样的缩写支持要求。
  • 运行时间：与指定 commit 含有文件数成线性，与 commit 总数无关。
  • 错误
    • 如果指定 id 的 commit 不存在，输出错误信息 No commit with that id exists.
    • 如果工作仓库中存在未跟踪文件会被 reset 命令覆写，输出错误信息 There is an untracked file in the way; delete it, or add and commit it first.
  • 代码行数：~10

merge [branch name]

  • 描述：将指定 id 的 branch 合并到当前 branch。
    1. 如果分裂点正是指定的 branch，直接输出 Given branch is an ancestor of the current branch. ；
    1. 如果分裂点就是当前所在 branch，则 checkout 指定 branch，输出Current branch fast-forwarded. ；
    1. 所有在分裂点之后的在指定 branch 中被修改、但是没有在当前 branch 修改的文件需要被转换成指定 branch 的版本，然后全部自动暂存；
    1. 在当前 branch 被修改、但是在指定 branch 中没有被修改的文件保持现状；
    1. 在两个 branch 中被以相同方式修改的文件保持现状；如果工作仓库中存在一个在不同分支均被移除的同名文件，继续保持未跟踪状态；
    1. 在分裂点未出现、只在当前 branch 出现的文件保持现状；
    1. 在分裂点未出现、只在指定 branch 出现的文件应该被 checkout 然后暂存；
    1. 在分裂点中出现、在当前 branch 未修改、在指定 branch 中不存在的文件应该被移除；
    1. 在分裂点中出现、在指定 branch 未修改、在当前 branch 中不存在的文件应该维持现状；
    1. 两个 branch 中以不同方式修改的同一文件存在冲突，存在冲突的情况：
      • 两个文件都被改变了内容，且彼此不同；
      • 其中一个文件的内容被改变，另一个文件被删除；
      • 文件在分支点不存在，在两个分支中有不同的内容。

      将存在冲突的文件内容替换为如下内容：

      <<<<<<< HEAD
contents of file in current branch
=======
contents of file in given branch
>>>>>>>

      接下来根据此前的暂存更新自动 commit ，commit message 为 Merged [given branch name] into [current branch name]. ；如果合并存在冲突，输出信息 Encountered a merge conflict. 合并的 commit 将两个被合并 branch 的 head 都记作父节点。

  • 运行时间：O(NlgN+D)，其中 N 为两 branch 祖先 commit 总数，D 为这些 commit 文件数据总量
  • 错误
    • 如果暂存区仍有更改，输出错误信息 You have uncommitted changes.
    • 如果给定 branch 不存在，输出错误信息 A branch with that name does not exist.
    • 如果尝试合并当前 branch，输出错误信息 Cannot merge a branch with itself.
    • 如果合并的 commit 没有任何修改导致错误，让 commit 输出它的错误信息
    • 如果一个当前 commit 未跟踪的文件会被合并覆写或删除，输出错误信息 There is an untracked file in the way; delete it, or add and commit it first.
  • 代码行数：~70
