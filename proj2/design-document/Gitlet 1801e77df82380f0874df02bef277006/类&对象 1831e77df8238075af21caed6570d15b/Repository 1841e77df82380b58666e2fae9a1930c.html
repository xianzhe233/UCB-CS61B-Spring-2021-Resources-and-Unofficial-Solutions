Repository

思路

  1. 包括工作目录和暂存区，通过对 commit，blob，branch 等对象操作，完成各种 command 需要的对工作目录和暂存区的基础操作；
  1. Repository 需要一个初始化方法，建立整个 .gitlet 仓库，包含需要的文件夹和文件，创建第一个 initial commit，然后初始化 branch 为 master，初始化 head 指向 master；
  1. .getlet 的结构如下：
    • .gitlet
      • blobs
        • blob…
      • commits
        • id head…
          • id tail…
      • branches
        • branch…
      • head
      • stagingArea
        • addition
        • removal
      • remote（EC）

    初始化时：

    1. 新建空 .gitlet 文件夹；
    1. 新建空 blobs 文件夹；
    1. 新建空 branches 文件夹；
    1. 新建空 head；
    1. 新建空 remote（序列化数据结构，EC）
    1. 新建 stagingArea 文件夹，新建 addition 和 removal 空文件（序列化空数据结构）；
    1. 创建 initial commit；
    1. 创建指向 initial commit 的 branch，branch 名为 master；
    1. 令 head 指向 master。
  1. Repository 主要处理 staging area 和 head，其中 staging area 包含了 addition 和 removal 两部分，分别使用 HashMap 和 HashSet 来保存被 add 的文件名与新 blob 文件的对应关系和被 rm 需要被取消跟踪的文件名。
  1. 在 stagingArea 中保存 addition 和 removal 对象的序列化，只在操作时反序列化取出使用；
  1. Repository 需要一个判断当前仓库是否存在的方法；
  1. Repository 需要一个扫描工作仓库文件的方法，在一些命令运行时确定未跟踪和已修改未暂存的文件；
  1. Repository 需要一个集成的 commit 方法，在创建新 commit 的同时更新 branch；
  1. Repository 需要支持获取 head 和修改 head；
  1. Repository 需要获取和修改 addition 和 removal；
  1. Repository 需要一个清空 staging area 的方法；
  1. Repository 不需要实例化，维护的内容都从对应文件中存取；

属性

  • 类变量 CWD
  • 类变量.getlet 文件
  • 类变量.getlet/stagingArea/addition 文件
  • 类变量.getlet/stagingArea/removal 文件

方法

  • static void init()
    初始化当前仓库。
  • static boolean exists()
    判断当前仓库是否存在。
  • static Commit getHead()
    从文件中获取 head 指向的 commit。
  • static void setHead(String branchName)
    修改 head 指向的 branch，保存到 head 文件中。
  • private static HashMap<String, String> getAddition()
    从文件中获取 addition 对象（HashMap）。
  • private static HashSet<String> getRemoval()
    从文件中获取 removal 对象（HashSet）。
  • static void removeFromAddition(String fileName)
    将指定的文件从 addition 中移除。
  • static void stagingAdd(String fileName, String blob)
    将 文件名-blob_id 键值对加入 addition 并保存至文件。
  • static void stagingRemove(String fileName)
    将指定文件加入 removal 并保存至文件。
  • static List<String> WDFiles()
    返回工作仓库中所有的文件。
  • static List<String> untrackedFiles(List<String> files)
    从给定的文件列表中找出未跟踪的所有文件，返回所有排序后的文件名。
  • static List<String> modifiedFiles(List<String> files)
    从给定的文件列表中找出
    修改未暂存的文件，返回所有排序后的文件名。
  • static void clearStagingArea()
    清除 staging area（将 addition 和 removal 都清空即可）。
  • static void commit()
    集成的
    commit 方法。
  • static void checkout(Commit commit, String fileName)
    从 commit 中取出文件名为 fileName 的文件 blob，覆写在工作仓库中。
