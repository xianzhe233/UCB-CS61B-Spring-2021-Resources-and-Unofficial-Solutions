Remote

思路

  1. Remote 表示一个 Gitlet 仓库连接的远程仓库的仓库名与地址对应关系，不需要实例化；
  1. Remote 表示的对应关系使用 HashMap，实例化文件保存于.getlet/remote；
  1. Remote 同样需要访问或修改仓库 head、branch、blob 的能力，但是需要访问的是远程仓库的；
  1. Remote 需要一个将某个 commit 从一个仓库复制到另外一个仓库的能力，具体来说：
    • 需要将指定的 commit 对应的文件复制过去；
    • 需要将指定的 commit 包含的文件对应的 blob 文件复制过去。

方法

  • static void initRemote()
    初始化远程仓库文件，将一个空 HashMap 写入
    .getlet/remote 文件。
  • static boolean exists(String remoteName)
    判断一个远程仓库是否存在。
  • static String geneticPath(String path)
    将给定的地址转化为在不同系统都能工作的地址。
  • private static File remoteRepo(String remoteName)
    返回指定远程仓库的文件目录。
  • private static File remoteCommitDir(String remoteName)
    返回指定远程仓库存放 commit 文件的目录。
  • private static File remoteBlobDir(String remoteName)
    返回指定远程仓库存放 blob 文件的目录。
  • private static File remoteBranchDir(String remoteName)
    返回指定远程仓库存放 branch 文件的目录。
  • static void addRemote(String remoteName, String address)
    新加入一个远程仓库，写入文件。
  • static void removeRemote(String remoteName)
    移除一个远程仓库，写入文件。
  • private static File getBlobFile(String remoteName, String blobId)
    获取指定远程仓库中的指定 blob 的文件。
  • private static File getBranchFile(String remoteName, String branchName)
    获取指定远程仓库中的指定 branch 的文件。
  • private static File getCommitFile(String remoteName, String commitId)
    获取指定远程仓库中的指定 commit 的文件。
  • static boolean hasBranch(String remoteName, String branchName)

     判断一个远程仓库中是否含有指定名称的 branch。
  • static Commit getBranch(String remoteName)
    获取指定远程仓库中的 branch 指向的 commit。
  • static Commit getCommit(String remoteName, String commitId)
    获取指定远程仓库中的 commit。
  • static void setBranch(String remoteName, String branchName, String commitId)
    将远程仓库中的指定 branch 指向指定 commit。
  • private static void copyCommit(File fromRepo, File toRepo, String commitId)
    将指定 id 的 commit 从 fromRepo 复制到 toRepo。
  • private static void copyBlob(File fromRepo, File toRepo, String blobId)
    将指定 id 的 blob 从 fromRepo 复制到 toRepo。
